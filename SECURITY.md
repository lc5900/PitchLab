# Security Policy

## Supported Versions

The project is early-stage. Security fixes target the latest `main` branch unless release branches are introduced later.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security reports.

Until a dedicated security contact is published, create a private GitHub security advisory if the repository is hosted on GitHub, or contact the maintainer through the repository owner profile.

## Privacy-Sensitive Areas

PitchLab uses microphone access. The intended behavior is:

- Microphone samples are processed locally.
- Raw audio is not saved.
- Practice history stores summary data only.

Please report any behavior that violates those expectations.
